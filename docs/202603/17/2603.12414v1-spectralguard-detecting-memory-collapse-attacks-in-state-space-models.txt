# Abstract 

State Space Models (SSMs) such as Mamba achieve linear-time sequence processing through input-dependent recur-rence, but this mechanism introduces a critical safety vul-nerability. We show that the spectral radius ρ( ¯A) of the discretized transition operator governs effective memory horizon: when an adversary drives ρ toward zero through gradient-based Hidden State Poisoning, memory collapses from millions of tokens to mere dozens, silently destroying reasoning capacity without triggering output-level alarms. We prove an evasion existence theorem showing that for any output-only defense, adversarial inputs exist that si-multaneously induce spectral collapse and evade detection, then introduce SpectralGuard , a real-time monitor that tracks spectral stability across all model layers. Spectral-Guard achieves F1 = 0 .961 against non-adaptive attackers and retains F1 = 0 .842 under the strongest adaptive setting, with sub-15ms per-token latency. Causal interventions and cross-architecture transfer to hybrid SSM-Attention systems confirm that spectral monitoring provides a prin-cipled, deployable safety layer for recurrent foundation models. 

# 1 Introduction 

Despite the rapid deployment of State Space Models (SSMs) [2–4] in production systems, we prove that no de-fense operating solely on model outputs can detect a class of attacks that silently destroys reasoning capacity from within. This impossibility—formalized in Theorem 3— arises because SSMs compress their entire history into a low-dimensional recurrent state ht ∈ Rd, updated at every token via a discretized transition operator ¯At = exp(∆ tA).An adversary who manipulates the input-dependent step size ∆t can drive the spectral radius ρ( ¯At) toward zero, col-lapsing effective memory from millions of tokens to mere dozens, while output distributions remain superficially plausible. We call this failure mode spectral collapse , and we build both the theory to explain it and the defense to stop it. The vulnerability is architectural. Unlike Transformers, which distribute information across an explicit key–value cache with additive attention aggregation, SSMs multiply by ¯At at every recurrent step. This creates a compound-ing attack surface: a small adversarial shift in ∆t cascades exponentially through recurrence ( 0.310 ≈ 6×10 −6), oblit-erating state information before any output-level alarm can fire. Hidden-state manipulation attacks (HiSPA) [ 5] ex-ploit precisely this pathway, steering internal recurrence into a high-contraction regime without triggering jailbreak detectors [ 6 , 7]. Our framework provides the first mecha-nistic explanation for why such attacks succeed on Mamba-based systems but not on comparable Transformers—a phenomenon previously observed but never grounded in theory. As SSMs enter safety-critical deployments, the mismatch between internal vulnerability and external ob-servability demands a principled response: monitoring the internal dynamics directly.

22.3 The HiPPO Framework and Memory Retention 

The HiPPO (High-order Polynomial Projection Operators) framework [ 8 ] provides a principled initialization for SSM matrices A that enables long-range memory. HiPPO ma-trices are designed to approximate continuous signals by projecting them onto orthogonal polynomial bases (e.g., Legendre polynomials). The key insight is that the contin-uous operator A should compress the input history into a finite-dimensional state ht such that reconstruction error is minimized. For the Legendre measure, the HiPPO matrix has eigen-values clustered near the imaginary axis with real parts Re (λi) ≤ 0, ensuring stability. After discretization via Eq. (1) , the discrete eigenvalues lie within or near the unit circle, with ρ( ¯A) close to but less than 1. This spectral configuration is necessary for long-context performance but, as we demonstrate, also creates an attack surface. 

## 2.4 Selective SSMs and the Mamba Archi-tecture 

Mamba [ 2 ] extends structured SSMs (S4 [ 3]) by introduc-ing input-dependent parameters ∆t, Bt, and Ct, computed via learned projections of the input token xt. This selec-tivity allows the model to decide per token how much to retain versus forget, analogous to gating mechanisms in LSTMs [12] but operating in continuous state space. Computational efficiency is achieved through a scan algorithm that parallelizes the recurrence during training (similar to parallel prefix sums) and a custom CUDA kernel for inference. The combination of selectivity and efficiency has enabled Mamba models to match or exceed Trans-former performance on long-context benchmarks [ 4, 13 ]while maintaining linear complexity in sequence length. However, the input-dependent discretization introduces a gradient pathway from token choice to spectral proper-ties: tokens → ∆t → ¯At → ρ( ¯At). An adversary can exploit this pathway to minimize ρ( ¯At), forcing the model into a high-contraction regime. 

## 2.5 State Space Models and Memory 

The modern SSM renaissance began with S4 [ 3 ], which paired HiPPO initialization [ 8] with diagonal-plus-low-rank parameterizations for O(N log N ) convolutions. Sub-sequent variants—S5 [ 34 ] (complex-to-real simplification), H3 [ 4] (data-dependent convolutions), and Hyena [ 13 ](implicit neural operators replacing SSMs)—explored dif-ferent trade-offs between expressiveness and efficiency. Mamba [ 2] unified these threads via selective discretiza-tion, achieving Transformer-competitive language model-ing at linear complexity. Concurrent architectures such as RetNet [ 14 ] (retention with exponential decay) and RWKV [ 15 ] (element-wise time-mixing) share the recur-rent memory paradigm but differ in how spectral prop-erties manifest: RetNet uses fixed decay rates without continuous-time foundations, while RWKV’s element-wise updates reduce the spectral attack surface. The classi-cal vanishing gradient problem in RNNs [ 12 , 35 ]—where 

|λi(W )| < 1 causes exponential gradient decay—is miti-gated in SSMs by HiPPO’s principled initialization ensur-ing ρ( ¯A) ≈ 1. Our Theorem 1 generalizes these classical analyses to selective SSMs with input-dependent ρ( ¯At).

## 2.6 Adversarial Attacks on Language Mod-els 

Adversarial ML spans gradient-based attacks in vision [ 16 ,17 ] and language [ 7, 38 ], with language-specific threats including jailbreaking [ 6, 37 ], prompt injection [ 18 , 54 ], universal adversarial suffixes [ 7 ], and character/word-level perturbations [ 39 , 40 ]. These attacks primarily aim to change output intent—forcing models to produce disal-lowed content. HiSPA [ 5] introduces a fundamentally distinct threat model: capability denial via internal state manipulation, analogous to denial-of-service attacks in cy-bersecurity where the system remains operational but loses critical reasoning capabilities. Our spectral framework pro-vides the first mechanistic explanation for why this attack class succeeds and the first principled defense. 

## 2.7 Defenses and Certified Robustness 

Output-based defenses [ 55 , 56 ] detect prohibited content via classifiers or perplexity heuristics but are blind to in-ternal degradation (formalized in Theorem 3). Certified defenses [ 41 , 42 ] provide provable guarantees via inter-val bound propagation or randomized smoothing [ 43 ], but face fundamental challenges in language: discrete input spaces, exponential output cardinality ( |V| L), and black-box prompting without gradient access. SpectralGuard complements these approaches by providing a deployable completeness guarantee (Theorem 4) operating on a contin-uous, low-dimensional internal quantity ( ρ), paired with a Lipschitz perturbation certificate (Theorem 5) that bounds adversarial spectral drift. 

## 2.8 Mechanistic Interpretability 

Mechanistic interpretability [ 19 , 20 ] decomposes neural networks into human-understandable circuits. For Trans-formers, key findings include induction heads [ 44 ], IOI circuits [ 45 ], copy-suppression [ 46 ], and superposition phe-nomena [ 47 ] where polysemantic neurons represent mul-tiple features. SSM interpretability remains nascent [ 48 ]. Our contribution identifies ρ( ¯At) as a monosemantic 

3quantity—a single interpretable scalar predicting memory capacity and reasoning performance—avoiding superposi-tion’s ambiguity and providing a compact O(1) diagnostic compared to attention pattern visualization at O(L2).

## 4.4 Threat Model 

We assume a white-box attacker with full access to model parameters (A, B, C, W∆) and the ability to compute gradients through the selective discretization pathway 

xt → ∆t → ¯At → ρ( ¯At). The attacker’s goal is to minimize ρ( ¯At), collapsing effective memory while keep-ing outputs superficially plausible. Adversarial tokens are constrained to the model vocabulary V (discrete space). The defender (SpectralGuard) has white-box access to in-ternal ¯At matrices during inference but no prior knowledge of attacker intent. We evaluate two attacker regimes: a non-adaptive at-tacker (unaware of SpectralGuard’s threshold ρmin ) and an adaptive attacker (aware of the threshold and attempt-ing evasion via gradient-optimized threshold proximity or multi-layer feature imitation).

Why a low linear R2 is expected. The multivariate

The low lin-ear R2 therefore does not weaken the mechanistic claim; it supports the phase-transition interpretation. The Pear-son correlation r = 0 .49 (p < 10 −26 ) remains highly significant as a monotone trend detector, and the mecha-nistic link is further confirmed by causal intervention (Sec-tion 5.4), where directly clamping ρ degrades performance with model weights frozen—ruling out confounding. 

Sources of remaining variance. Beyond the phase-transition nonlinearity, residual variance is governed by non-scalar dynamics: (1) the direction of the hidden state 

ht relative to the output projection C, which allows ad-versarial perturbations to bypass contraction if aligned with high-magnitude eigenvectors; (2) the non-linear in-teraction within the architecture’s interleaved components, which single-layer ¯A analysis ignores; and (3) the input-dependent nature of projections ∆, B, C . Furthermore, in 

short-context tasks, early semantic interactions dominate before the asymptotic decay rate ρ fully governs the state, suggesting ρ becomes a stricter bottleneck specifically for 

long-context reasoning where pure memory retention is decisive.

## 5.7 Experiment 7: Layer-Wise Collapse Analysis (Mechanistic Interpretability) 

To provide visual and mechanistic intuition into the dy-namic progression of the adversarial spectral state attack, we extract the sequence-wise minimum spectral radius ¯ρ

simultaneously across all 24 layers of the Mamba-130M model. Our layer-wise tracking establishes a clear bottleneck-ing dynamic. As illustrated in Figure 7, the initial layers 13 (0–3) maintain high spectral resilience near the pre-trained capacity ( ρ ≈ 0.95 ). However, the adversarial tokens (red) forcibly collapse the spectral radius primarily in the mid-to-early layers (specifically layers 4–10). When ¯ρ sinks systematically under the critical threshold ρmin = 0 .30 

within these layers, the continuous flow of contextual rea-soning is decisively bottlenecked. The later layers attempt to adapt, yet suffer from unrecoverable context starvation caused by the upstream spectral shrinkage. Conversely, benign flows (green) remain completely bounded well above the safe limit globally. This multi-layer interpretability mechanically demonstrates why the Multi-Layer SpectralGuard model achieves exceptional robust performance: the spectral decay attack creates a distinct, highly-structured depth-wise signature that non-malicious text generation cannot emulate.

# 6 Discussion 

## 6.1 Causal Mechanism 

Direct intervention on spectral dynamics confirms a mechanism-level causal link between spectral stability and reasoning capacity: clamping ρ( ¯At) degrades memory even when model weights are fixed, ruling out confounding by weight statistics alone (full methodology and threshold-wise results in Section 5.4). 

Reconciling theoretical and empirical horizons. The-orem 1 predicts Heff ≈ 1.17 × 10 6 tokens for ρ = 0 .98 ,while empirical performance degrades within ∼10 3 tokens. We view this gap not as a weakness of the theory but as an expected consequence of three factors: the bound is a worst-case ceiling assuming stationary ρ (Remark 2); in practice, input-dependent discretization causes ρ to fluctu-ate token-by-token, accelerating information loss; and task-specific capacity limits (e.g., binding slots, output rank) impose tighter bottlenecks than the information-theoretic bound alone. Critically, the claim that ρ causes memory loss does not rest on the absolute value of Heff but on 

triangulated evidence: (i) the theoretical bound guaran-tees that Heff must shrink when ρ drops—a qualitative prediction confirmed empirically; (ii) the phase transition at ρcritical ≈ 0.90 (Figure 2) shows that performance col-lapses abruptly when ρ crosses a threshold, matching the Corollary 2 prediction of O(1 /η ) scaling; and (iii) causal intervention experiments (Section 5.4) demonstrate that directly clamping ρ suffices to degrade performance with weights frozen. This three-pronged argument—bound, phase transition, causal intervention—establishes the spec-tral radius as a mechanistic control variable for memory, even though the bound itself is quantitatively loose. 

## 6.2 Implications for AI Safety and Deploy-ment 

SSMs are not inherently safer than Transformers. The crit-ical vulnerability lies in state safety : when internal recur-rence provides the computational substrate for reasoning, an adversary can silently degrade capability by manipu-lating spectral properties without triggering output-level alarms. This differs fundamentally from conventional jail-break scenarios—here, the attack destroys the mechanism 

supporting reasoning itself. 

Why SSMs fail where Transformers resist. Transform-ers distribute information across explicit key–value pairs with additive attention aggregation. To collapse memory, an attacker must corrupt the entire KV cache—a target spanning hundreds of megabytes that manifests immedi-ately in output distributions. SSMs, by contrast, compress history into a dstate -dimensional vector ( dstate = 16 for Mamba), updated multiplicatively by ¯At at every step. A single adversarial token can steer ¯At into a contracting regime, and this contraction compounds : 0.310 ≈ 6×10 −6,obliterating state information. This explains empirical ob-servations [ 5 ] that Mamba-based systems are more vulner-able to certain attacks, and identifies the gradient pathway from tokens to spectral properties as the responsible chan-nel. 

Regulatory alignment. The EU AI Act [ 27 ] mandates robustness assessments against adversarial inputs for high-risk AI systems. SpectralGuard provides a concrete com-pliance artifact: logged spectral radius traces constitute interpretable evidence of internal state integrity and enable forensic incident response. At <15ms/token multi-layer 16 latency, serving 1M requests/day at 200 tokens/response adds ≈ 0.02% of total inference FLOPs—negligible in production. 

## 6.3 Adaptive Attacks and the Arms Race 

Spectral monitoring is not immune to adaptive attack-ers, but the attacker’s optimization space is fundamentally constrained. Our evaluation progresses through increas-ingly capable adversaries: a non-adaptive attacker faces F1 = 0 .961 detection; a threshold-evasion attacker that spoofs single-layer ρ reduces this to F1 = 0 .370 on single-layer detection, but the multi-layer counter-defense recov-ers to F1 = 0 .950 ; and a multi-layer-aware attacker (Adap-tiveHiSPA) that jointly imitates benign spectral statis-tics ( μρ, σρ, inter-layer gap gρ) still faces F1 = 0 .842 

(AUC = 0 .903 ) detection. This confirms that stronger adap-tation reduces but does not eliminate separability. The fundamental constraint is Theorem 1: any attack that collapses memory must drive ρ( ¯At) below the critical threshold, and any evasive strategy that preserves near-critical ρ necessarily preserves memory—defeating the at-tack’s own purpose. This creates an inherent trade-off anal-ogous to the impossibility of perfect stealth in encryption backdoors [ 33 ]. Future threat vectors include directional attacks manipulating eigenvector alignment while preserv-ing mean ρ, and slow spectral decay attacks spreading collapse over hundreds of tokens; these can be countered by monitoring condition number κ( ¯At) and expanding the sliding window, respectively. Lexical-stealth constrained attackers face an additional Pareto frontier. We tested three attacker families (Lexical-only, Joint-Loss, and Random/Baseline) under varying regularization strengths ( λ). Figure 9 and Table 13 (Ap-pendix A.5) illustrate the structural trade-off: no optimiza-tion strategy achieves simultaneous lexical indistinguisha-bility ( AUC lex ≈ 0.50 ) and measurable spectral damage (∆ρmean > 0.02 ). This trade-off is governed by a topo-logical lock : achieving lexical stealth structurally requires the input sequence to remain near the benign data mani-fold, which naturally conditions the SSM representation geometry to preserve near-unit eigenvalue magnitudes. Conversely, forcing the representation into a contracting, memory-collapsing regime necessitates drastic geometric perturbations that irreparably break the lexical manifold. Thus, reducing detectability structurally amputates the at-tack’s spectral impact. 

Quantifying the stealth-damage budget. Table 13 re-veals a sharp asymmetry: the Joint-Loss family achieves 

∆ρmean = 0 .557 at λ = 1 .0 but only by surrendering lexi-cal invisibility to AUC lex = 0 .701 , while the Lexical-only family sustains AUC lex > 0.94 at the cost of negligible damage ( ∆ρmean ≤ 0.037 ). Random/Baseline probes clus-ter in the lower-left quadrant, confirming that significant spectral harm requires deliberate, structure-aware gradi-ent steering—not merely unconstrained token selection. These results formalize a stealth-damage budget : each unit of spectral impact an attacker acquires trades com-pounding surface detectability, with the frontier governed by the Lipschitz constant of the input-to-eigenvalue map (Theorem 5). 

## 6.4 Limitations 

Limitation 1: Multi-layer overhead. Multi-layer moni-toring adds a stable ∼15% throughput penalty, acceptable for most deployments but non-trivial for latency-critical systems. Kernel-fused extraction (discretization and radius estimation fused per block) is a concrete path to reducing this below 10%. 

Limitation 2: Hybrid recovery dynamics. In Zamba2, attention pathways can partially mask downstream effects of SSM collapse, making severity on final outputs harder to quantify than in pure SSMs. Controlled clamp-release experiments measuring time-to-recovery would clarify this interaction. 

Limitation 3: Adaptive attacker escalation. White-box attackers can optimize near-threshold behavior. The Lips-chitz certificate (Theorem 5) bounds minimum detectable spectral drift, but stronger adaptive evaluations with tem-poral smoothing are needed to certify robustness under escalation. 

Limitation 4: Task-domain concentration. Current benchmarks emphasize memory-heavy tasks. Domain-stratified calibration across reasoning, coding, narrative, and dialogue—with per-domain operating thresholds—is needed to validate generalization. 

## 6.5 Broader Impact 

This work targets deployment safety for recurrent foun-dation models. SpectralGuard enables auditable, mecha-nistic monitoring for capability degradation and incident response in high-stakes settings. Misuse risk—attackers reverse-engineering spectral thresholds—is mitigated by our adaptive evaluations, explicit limitation disclosure, and recommendation of randomized thresholding with periodic retraining in production. ## 6.6 Future Directions 

The spectral monitoring paradigm opens several high-impact research directions. 

Resolving the lexical-stealth frontier. Our results estab-lish that attackers face a Pareto trade-off between lexical evasion and spectral damage. Systematic exploration of this frontier across diverse attacker families will reveal whether the trade-off is fundamental or an artifact of cur-rent optimization strategies, potentially yielding certifiable stealth-resistance bounds. 

Adversarial robust training. Retraining the detector with frontier adversarial prompts and calibrated thresholds can close the gap between non-adaptive (F1 = 0 .961 ) and adaptive (F1 = 0 .842 ) performance, strengthening Spec-tralGuard without sacrificing deployment simplicity. 

Cross-architecture generalization. Extending the framework to additional SSM and hybrid architectures (Mamba-2 variants, Jamba [ 29 ]) will test whether spectral collapse signatures are universal properties of recurrent computation, as our Zamba2 transfer results suggest. 18 Efficient production monitoring. Monitoring a subset of critical layers (informed by our layer-wise collapse anal-ysis showing layers 4–10 as primary bottlenecks) com-bined with temporal smoothing can preserve ≥95% of detection performance while reducing runtime overhead below 10%. 

## 6.7 Extended Related Work 

6.7.1 Structured State Space Models 

The modern SSM renaissance began with S4 [ 3 ], which introduced HiPPO initialization and diagonal-plus-low-rank (DPLR) parameterizations enabling O(N log N ) con-volutions via FFTs. Subsequent work (S5 [ 34 ], H3 [ 4 ], Hyena [13]) explored variations: • S5 simplified S4 via complex-to-real transformations. • H3 introduced data-dependent convolutions. • Hyena replaced SSMs with implicit neural operators. Mamba [ 2 ] unified these threads via selective discretiza-tion, achieving Transformer-competitive performance on language modeling while maintaining linear complexity. To our knowledge, our work is among the first to ana-lyze Mamba’s adversarial properties through the lens of spectral theory. 

6.7.2 Memory and Long-Range Dependencies 

The vanishing gradient problem in RNNs [ 12 , 35 ] stems from eigenvalue decay: when |λi(W )| < 1 for the weight matrix W , gradients vanish exponentially over time. LSTMs [ 12 ] and GRUs [ 36 ] mitigate this via gating, effec-tively learning when to preserve ρ(W ) ≈ 1.HiPPO [ 8] provides a principled alternative by construct-ing A to approximate continuous signals via orthogonal polynomials, ensuring ρ( ¯A) ≈ 1 after discretization. Our Theorem 1 generalizes classical vanishing gradient analy-ses to selective SSMs with input-dependent ρ( ¯At).

6.7.3 Adversarial Robustness in NLP 

Adversarial NLP spans multiple threat models: • Character/word perturbations [39 , 40 ]: Swapping to-kens to fool classifiers. • Universal adversarial triggers [ 38 ]: Fixed phrases causing misclassification. • Gradient-based attacks on embeddings [ 7]: Optimiz-ing continuous embeddings projected to tokens. • Jailbreaks [6, 37 ]: Prompt engineering to bypass safety filters. HiSPA [ 5 ] introduces a distinct threat: capability denial 

via state manipulation. Our spectral framework provides the first mechanistic explanation and defense for this attack class. 

6.7.4 Defenses and Certified Robustness 

Certified defenses [ 41 , 42 ] provide provable robustness guarantees, typically via interval bound propagation or randomized smoothing [ 43 ]. For language models, certifi-cation is challenging due to: • Discrete input space (tokens) vs. continuous (images). • Exponential output space ( |V| L sequences of length L). • Black-box prompting: No access to internal gradients. SpectralGuard provides a deployable guarantee (The-orem 4) and a perturbation certificate (Theorem 5) via Lipschitz continuity of ∆ 7 → ρ(exp(∆ A)) .

6.7.5 Interpretability and Causal Analysis 

Mechanistic interpretability [ 19 , 20 ] decomposes models into human-understandable circuits. For Transformers, key findings include: • Induction heads [ 44 ]: Two-layer circuits copying tokens. • IOI circuits [ 45 ]: Name disambiguation via positional attention. • Superposition [ 47 ]: Polysemantic neurons representing multiple features. For SSMs, interpretability is nascent. Our contribu-tion identifies ρ( ¯At) as a monosemantic quantity—a single interpretable scalar predicting behavior—avoiding super-position’s ambiguity. 

6.7.6 Control Theory Meets Deep Learning 

Control-theoretic perspectives on neural networks [ 49 , 50 ]model ResNets as discretizations of ODEs. Stability analyses [ 51 , 52 ] constrain ∥∇ hf ∥ ≤ 1 to prevent divergence. For SSMs, stability is encoded in 

ρ( ¯A) < 1, providing a natural control-theoretic lens. Reachability and controllability [ 9 ] quantify whether all states are reachable from inputs. Our adversarial frame-work inverts this: attackers reduce controllability by col-lapsing ρ( ¯A), shrinking the reachable set. 

# 7 Conclusion 

We have established that the spectral radius of the dis-cretized transition operator is both the mechanism enabling long-range reasoning in State Space Models and the attack surface through which adversaries can silently destroy it. 19 The Spectral Foundations framework formalizes this dual-ity: a tight horizon bound explains when and why SSMs forget, an evasion existence theorem proves that output-only defenses face fundamental limitations, and Spectral-Guard demonstrates that direct spectral monitoring is both necessary and sufficient for real-time defense. Our results hold across model scales, adaptive threat models, and hy-brid architectures, providing the first mechanistic security guarantee for recurrent foundation models. As these ar-chitectures move from research prototypes to production infrastructure, spectral monitoring offers what gradient clipping once provided for stable training: a lightweight, principled safeguard for the internal dynamics that make reasoning possible. The era of recurrent foundation models demands recurrent vigilance.

## A.5 Additional Analyses 

Layer-wise spectral radius distribution. We measure 

ρ( ¯At) across all 24 layers for benign vs. adversarial prompts. Findings: • Benign: ρ relatively uniform ( 0.96 ± 0.03 across layers) • Adversarial: Early layers (1–8) show sharp drop ( ρ ≈

0.32 ), later layers less affected ( ρ ≈ 0.65 )• Implication: Monitoring layer 1–8 suffices for detection; late layers provide redundancy 

Transferability across model sizes. In a separate transfer-only stress test (distinct from the primary 

mamba-130m-hf benchmark), we optimize adversarial prompts on Mamba-370M and test transfer to Mamba-1.4B and Mamba-2.8B: • Transfer accuracy drop: 370M →1.4B: −38 .2% (down from −52 .5% )• Transfer accuracy drop: 370M →2.8B: −28 .7% 

• Conclusion: Attacks partially transfer but lose potency with scale (likely due to increased capacity providing redundant memory paths) 

Transformer Proxy Baseline (GPT-2). We ran a paired baseline protocol for GPT-2 small under HiSPA-style opti-mization to test architecture specificity. The protocol mir-rors prompt splits and attack budget, while replacing SSM spectral probes with attention/hidden-state proxy stability features and honest train/test separation. The proxy met-rics were fully saturated (AUC/F1 = 1.00), while lexical control also saturated at AUC=1.00. Therefore, this exper-iment was interpreted strictly as evidence of unresolved lexical leakage in the prompt family, not as architecture-level proof about Transformer resilience or immunity. 24
